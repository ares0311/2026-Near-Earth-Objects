# HAZARD_ASSESSMENT_GUIDE.md — Hazard Assessment Technical Reference

This document describes the `HazardAssessment` schema model, the hazard-flag
taxonomy, the alert-pathway decision tree, PHA criteria, orbit-quality
requirements, and the mandatory guardrail that the pipeline does NOT
autonomously assert impact probabilities.

---

## HazardAssessment Fields

| Field | Type | Description |
|---|---|---|
| `hazard_flag` | `HazardFlag` | Primary hazard classification for this candidate |
| `moid_au` | `float \| None` | Minimum Orbit Intersection Distance in AU; `None` when orbit quality is too low to compute |
| `estimated_diameter_m` | `float \| None` | Estimated diameter in metres from H-magnitude (albedo p_v = 0.14); `None` if H unknown |
| `absolute_magnitude_h` | `float \| None` | Absolute magnitude H; `None` if photometry is insufficient |
| `neo_class` | `NEOClass` | Dynamical class: `amor`, `apollo`, `aten`, `ieo`, or `unknown` |
| `alert_pathway` | `AlertPathway` | Recommended external reporting action |
| `explanation` | `CandidateExplanation` | Human-readable classification breakdown |
| `orbital_elements` | `OrbitalElements \| None` | Best-fit orbital elements; `None` for short-arc objects |

---

## HazardFlag Values

| Value | Meaning |
|---|---|
| `pha_candidate` | MOID ≤ 0.05 AU AND H ≤ 22 AND orbit quality ≥ 2 |
| `close_approach` | MOID ≤ 0.05 AU but H > 22 or orbit quality < 2 |
| `nominal` | No Earth-threatening characteristics detected |
| `unknown` | Insufficient data to assess hazard status |

The pipeline assigns `pha_candidate` only when ALL three conditions are met:
1. MOID ≤ 0.05 AU (computed by `orbit.compute_moid`)
2. Absolute magnitude H ≤ 22 (corresponds to diameter ≳ 140 m at p_v = 0.14)
3. Orbit quality code ≥ 2 (multi-night arc; single-night arc gives unreliable MOID)

## AlertPathway Decision Tree

Candidates are routed to exactly one alert pathway through an ordered gate:

```
1. known_object      — MPC cross-match distance < threshold → stop
2. internal_candidate — real_bogus_score < 0.65 or arc < 1 night → stop
3. neocp_followup    — object appears on NEOCP → request follow-up
4. mpc_submission    — new detection; submit to MPC for confirmation
5. nasa_pdco_notify  — PHA candidate WITH independent MPC confirmation
```

The `nasa_pdco_notify` pathway is only reachable after:
- MPC submission (Step 1 of alert protocol)
- Independent confirmation by ≥ 2 observatories OR ≥ 24-hour NEOCP monitoring
- CNEOS Scout/Sentry assigns impact probability ≥ 0.01 %

## PHA Criteria

A Potentially Hazardous Asteroid (PHA) candidate must satisfy:

| Criterion | Threshold | Source |
|---|---|---|
| MOID | ≤ 0.05 AU | `orbit.compute_moid` |
| Absolute magnitude H | ≤ 22.0 | Corresponds to diameter ≳ 140 m |
| Orbit quality code | ≥ 2 (multi-night arc) | `orbit.arc_quality_report` |
| Not a known MPC object | `known_object_score` < 0.8 | `classify` stage |

Failing any criterion downgrades the candidate to `close_approach` or `nominal`.

---

## Orbit Quality Requirement

MOID values are unreliable for arcs shorter than one full night. The pipeline
enforces orbit quality code ≥ 2 before assigning `pha_candidate`. Quality
codes are defined as:

| Code | Arc | Description |
|---|---|---|
| 1 | < 1 day | Single-night arc; MOID estimate unreliable |
| 2 | Multi-night | MOID usable; orbit still poorly constrained |
| 3 | ≥ 1 week | Well-constrained preliminary orbit |
| 4 | Opposition | High-quality orbit with opposition coverage |

## Guardrail — No Autonomous Impact Claims

**The pipeline does NOT autonomously assert any impact probability.**

All hazard outputs from this pipeline are preliminary candidate assessments
intended for human review and external verification. The pipeline:

- Does NOT compute absolute impact probabilities
- Does NOT publish or announce hazard findings autonomously
- Does NOT trigger the `nasa_pdco_notify` pathway without independent
  observatory confirmation and CNEOS Scout/Sentry assessment
- Always defers final impact-probability statements to MPC/CNEOS

Every alert package generated by `alert.py` includes a mandatory guardrail
statement that must contain the word "NOT" to confirm compliance.

See `docs/ALERT_PROTOCOL.md` for the complete mandatory alert decision tree
and the step-by-step guardrail checklist.
